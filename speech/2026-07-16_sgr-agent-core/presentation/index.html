Проблема

Free-form prompting непредсказуем

  • Агент сам решает, когда рассуждать, а когда звать инструмент
  • Пропускает шаги рассуждения и путает их порядок
  • Ход мыслей спрятан внутри модели, повторить и отладить нельзя

Промпт просит модель быть аккуратной, но ничего не гарантирует

Решение

SGR переводит рассуждение в структуру данных

  • Рассуждение описано как строгая pydantic-схема
  • Модель обязана заполнить все поля по порядку, прежде чем действовать
  • Пропустить шаг или переставить его нельзя - это зашито в ядро фреймворка

Не просим модель быть умной, а даём рельсы, по которым думать

Базовый компонент - схема

Та самая структура в коде

class ReasoningTool(SystemBaseTool):
    reasoning_steps:   list[str]   # ход мысли, 2-3 шага
    current_situation: str         # где мы сейчас
    plan_status:       str         # статус плана
    enough_data:       bool        # хватает данных
    remaining_steps:   list[str]   # что осталось
    task_completed:    bool        # задача закрыта

Каждое поле это обязательный шаг рассуждения, а не пожелание в промпте

  • Routing это выбор следующего инструмента на основе заполненной схемы
  • Есть лимиты итераций и кларификаций, чтобы агент не крутился вечно
Как это в коде - свой агент

Агент с нуля через наследование

from sgr_agent_core.agents import ToolCallingAgent

class SGRFileAgent(ToolCallingAgent):
    name = "sgr_file_agent"

    # подключаем свои тулы для работы с файлами
    toolkit = [ListDirectoryTool, ReadFileTool,
               SearchInFilesTool, FindFilesFastTool]

Наследуемся от готового типа и добавляем свои инструменты, ядро уже умеет рассуждать

Как начать

Установка, конфиг, запуск

# 1. поставить пакет
pip install sgr-agent-core

# 2. описать llm, агентов и тулы в config.yaml, затем запустить
sgr    -c config.yaml     # HTTP-сервер, OpenAI-совместимый API
sgrsh  -c config.yaml     # интерактивный CLI
sgracp -c config.yaml     # агент по протоколу ACP через stdio

Дальше обращаемся как к обычному OpenAI API, где model это имя агента

Итог

Зачем начинать пользоваться

  • Предсказуемость рассуждение по схеме, без пропущенных шагов
  • Переносимость любой OpenAI-совместимый провайдер, включая локальные модели
  • Расширяемость свои тулы, схемы и агенты через наследование и конфиг
