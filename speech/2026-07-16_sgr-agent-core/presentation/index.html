Проблема

Free-form prompting непредсказуем

  • Агент сам решает, когда рассуждать, а когда звать инструмент
  • Пропускает шаги рассуждения и путает их порядок
  • Ход мыслей спрятан внутри модели, повторить и отладить нельзя

Промпт просит модель быть аккуратной, но ничего не гарантирует

Как это выглядит

Один вопрос, два поведения

Free-form агент
userСравни цены на BMW X6 2025 в трёх странах
agentсразу зовёт поиск с общим запросом, без плана
agentотвечает по одной стране, две подзадачи потеряны
SGR агент
userСравни цены на BMW X6 2025 в трёх странах
reasoningplan - три подзадачи, по стране на каждую
reasoningenough_data=false - идём дальше, пока не собрано всё

Слева результат плавает от запуска к запуску, справа шаги фиксированы схемой

Решение

SGR переводит рассуждение в структуру данных

  • Рассуждение описано как строгая pydantic-схема
  • Модель обязана заполнить все поля по порядку, прежде чем действовать
  • Пропустить шаг или переставить его нельзя - это зашито в ядро фреймворка

Не просим модель быть умной, а даём рельсы, по которым думать

Базовый компонент - схема

Та самая структура в коде

class ReasoningTool(SystemBaseTool):
    reasoning_steps:   list[str]   # ход мысли, 2-3 шага
    current_situation: str         # где мы сейчас
    plan_status:       str         # статус плана
    enough_data:       bool        # хватает данных
    remaining_steps:   list[str]   # что осталось
    task_completed:    bool        # задача закрыта

Каждое поле это обязательный шаг рассуждения, а не пожелание в промпте

Конфиг - структура

Один config.yaml на всё

llm:              # модель и доступ
  model: "gpt-4o-mini"
  base_url: "https://api.openai.com/v1"
execution:        # лимиты цикла
  max_iterations: 10
  max_clarifications: 3
mcp:              # внешние источники по MCP
  mcpServers: {deepwiki: {url: "https://mcp.deepwiki.com/mcp"}}
tools: ...        # настройки тулов
agents: ...       # определения агентов

Секции llm и execution наследуются всеми агентами по умолчанию

Создание агента - через конфиг

Агент это несколько строк YAML

agents:
  fast_research_agent:
    base_class: "SGRToolCallingAgent"   # тип из коробки
    llm:
      model: "gpt-4o-mini"              # переопределяем только нужное
    tools:                            # набор инструментов
      - web_search_tool: {max_searches: 3}
      - "create_report_tool"
      - "final_answer_tool"

Выбираем базовый класс и список тулов - остальное берётся из глобального конфига

Создание агента - через код

Когда нужна своя логика - наследование

from sgr_agent_core.agents import ToolCallingAgent

class SGRFileAgent(ToolCallingAgent):
    name = "sgr_file_agent"
    toolkit = [ListDirectoryTool, ReadFileTool,
               FindTool, GrepTool]

    # при желании переопределяем шаги цикла

Наследник регистрируется в ядре автоматически и доступен по имени в конфиге

Создание тула

Тул это схема плюс один метод

class FindTool(BaseTool):
    tool_name = "find_tool"
    reasoning: str = Field(description="зачем ищем файлы")
    pattern:   str = Field(description="glob-паттерн имени")

    async def __call__(self, context, config) -> str:
        # своя логика, возвращаем текст для модели
        return find_files(self.pattern)

Поля Field это одновременно аргументы и подсказка модели, как их заполнить
