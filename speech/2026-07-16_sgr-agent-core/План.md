## Про проект коротко

SGR Agent Core - открытый агентный фреймворк на Schema-Guided Reasoning, Python, лицензия MIT. Репозиторий github.com/vamplabAI/sgr-agent-core, чат сообщества t.me/sgragentcore. Проект командный, делает сообщество neuraldeep и vamplabAI при поддержке red_mad_robot, концепция SGR из сообщества LLM Under the Hood и @abdullin. Роль Павла - разработка API. Про red_mad_robot и команду упоминаем на слайде о себе.

## Как собран доклад

Доклад чисто теоретический, без практических занятий. Осталась вся теория и компоненты - что такое SGR, архитектура (API, ACP, CLI), системные тулы и агенты, YAML-конфигурация с реестрами, создание своего тула и агента, запуск, два готовых примера (Deep Research и файловый агент), интеграции и метрики. Всего 24 слайда.

Все диаграммы нарисованы нативно через HTML и CSS в цветах презентации, картинки не вставлялись - схема SGR (чёрный ящик против прозрачного агента), связь компонентов (агент, тулы, MCP) и два пайплайна (Deep Research и файловый агент). Термин free-form prompting в докладе не используется.

## Учтены свежие фичи из репозитория

Мультипровайдерный веб-поиск (Tavily, Brave, Perplexity) и пагинация, тул выполнения кода в песочнице, агент iron_agent, обновлённый Langfuse, отмена выполнения агента. Перед выступлением сверить список тулов и агентов с актуальным README.

## Что обязательно проговорить

Четыре мысли, которые должны остаться у зала. Обычный агент это чёрный ящик, SGR делает рассуждение прозрачным и повторяемым через схему. Из коробки набор тулов и несколько типов агентов, всё собирается конфигом, а не кодом. Расширять просто - тул это схема плюс метод, агент это наследование плюс список тулов. Свой фронт писать не нужно - подключаемся готовыми клиентами через OpenAI API или ACP.

## Про слайды с кодом и схемами

Схемы на слайдах 4, 15, 19, 20 нарисованы нативно, читаются как обычный текст. Живой код на слайдах 5, 12, 13, 14, 16, 17 - читаем по смыслу, не построчно. ## Подготовка к вопросам

Чем SGR лучше обычного tool calling. Рассуждение становится видимым и повторяемым, а решение думать или звать инструмент принимается по схеме, а не на усмотрение модели. Цена - иногда лишние токены на фазу рассуждения.

Не дорого ли по токенам. У SGR-агентов есть отдельная фаза рассуждения. Поэтому есть несколько типов агентов - под скорость tool_calling_agent, под контроль sgr_agent, посередине гибрид, для простого iron_agent.

Работает ли на локальных моделях. Да, любой OpenAI-совместимый эндпоинт, в бенчмарках для этого есть локальный vLLM.

Насколько сложно написать своего агента. Три уровня. Агент строкой в конфиге. Своя логика - наследование, слайд 17. Свой тул - класс со схемой и одним методом, слайд 16. Всё регистрируется в ядре автоматически.

Какие тулы из коробки. Системные для управления ходом, поиск с провайдерами Tavily, Brave, Perplexity, извлечение контента, запуск команд и выполнение кода, файловые, создание отчётов.

Что такое ACP. Agent Client Protocol, общение редакторов с агентом по stdio через JSON-RPC. Бинарь sgracp поднимает того же агента с тем же конфигом.

Это уже в проде или в разработке. ACP и sgracp влиты в main, тесты и линтер зелёные. Проект в статусе alpha, API ещё может меняться, но фичи доступны из коробки.

Есть ли метрики качества. На бенчмарке SimpleQA около 86 процентов на gpt-4.1-mini, прогоны матрицей в CI, числа в README.
