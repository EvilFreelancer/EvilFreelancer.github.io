# Ссылки и источники

Сюда сложены все ссылки, которые цитирую и показываю на стриме про harness, см. план в [[Стрим про harness]]. Этот же список пойдёт в основу презентации. У части пунктов рядом стоит пометка, в каком блоке плана они нужны.

## Теория надёжности (блок 1)

- Lusser's law (закон Лессера), product probability law of series components, надёжность последовательной системы равна произведению надёжностей звеньев, отсюда система слабее самого слабого звена - https://en.wikipedia.org/wiki/Lusser's_law

## История и определения термина (блок 2)

Корень термина и линия eval harness:
- EleutherAI, lm-evaluation-harness, фактический стандарт для бенчмарков моделей - https://github.com/EleutherAI/lm-evaluation-harness
- OpenAI Evals, фреймворк для прогона моделей по сценариям - https://github.com/openai/evals

Где инженеры OpenAI используют слово harness в агентном смысле:
- OpenAI, Introducing SWE-bench Verified, разделение агентного harness и модели - https://openai.com/index/introducing-swe-bench-verified/
- OpenAI o1 System Card, оценки на разных scaffold и harness - https://arxiv.org/abs/2412.16720
- OpenAI GPT-5 System Card - https://arxiv.org/abs/2601.03267

Оформление harness в отдельную дисциплину в 2026:
- Viv Trivedy, Anatomy of an Agent Harness, формула agent = model + harness - https://x.com/Vtrivedy10/status/2031408954517971368
- Addy Osmani, Agent Harness Engineering - https://addyosmani.com/blog/agent-harness-engineering/
- O'Reilly Radar, Agent Harness Engineering - https://www.oreilly.com/radar/agent-harness-engineering/
- Hugging Face, разбор терминов harness и scaffold - https://huggingface.co/blog/agent-glossary
- awesome-harness-engineering, подборка инструментов и паттернов - https://github.com/ai-boost/awesome-harness-engineering

## Заметки по достоверности

- слово harness старше ИИ, корень из тестирования софта, test harness, поэтому в блоке 2 честнее говорить, что инженеры OpenAI не придумали слово, а первыми громко применили его к агентам;
- авторство дисциплины harness engineering чаще всего приписывают Viv Trivedy, точные даты статей стоит сверить по ссылкам перед эфиром;
- ссылки на Claude Code, Codex и Cursor ведут на официальные страницы продуктов, их стоит открыть один раз заранее, адреса иногда меняются.
